Ancillary Procedure (108_1) Schema Properties:


This file holds those procedures which are not included in the files
of the automated ancillaries. For example, respiratory therapy and
cardiology orders (EG. EKG, Pulmonary Function tests, Consultations)
These Orders may either be scheduled procedures or not. If
scheduled, they will go through PAS and the procedure must be
part of a clinic known to PAS.

OrderLabelTypeDescription
1Name (name-108_1) STRINGRequired field - free text - enter 3-30 characters. This field contains
the name of the ancillary procedure, including diet consults (i.e., calorie
count and nutritional assessment).
2Clinic (clinic-108_1) POINTER (Hospital_Location-44)Field is a pointer to the Hospital Location File (#44).
This field contains the name of the clinic associated with this ancillary
procedure.
3Schedule Type (schedule_type-108_1) ENUMERATION This field determines which edit template will be used for this procedure.
 CONTINUOUS; ONE TIME; SCHEDULED; PRN
4Mnemonic (mnemonic-108_1) STRINGEnter an abbreviation for easy lookup of this procedure.
5Service (service-108_1) POINTER (Department_And_Service-45_7)Field is a pointer to the Department and Service File (#45.7).
This field contains the name of the hospital service associated with this
clinic. For example, the Internal Medicine Clinic falls under the Medicine
service.
6Device (device-108_1) STRINGThe ID of the device where this order should be printed for
communication with the appropriate non-automated ancillary.
In the case of a procedure associated with a clinic, this
field is unnecessary since PAS handles the communication.
7Result Type (result_type-108_1) ENUMERATION This field defines the type of result for the procedure. Enter
"F" if the result of the procedure is going to be entered in free
text. Enter "S" if the result is going to be entered from a set of
codes. Enter "N" if the result is of numeric value. Enter "P" if
the procedure is made up of a panel of procdures; each has a result.
The panel of procedures is defined in the "RELATED PROCEDURES" field.
 FREE TEXT; SET OF CODES; NUMERIC; PANEL; COMPOUND
8Unavailable (unavailable-108_1) BOOLEANEnter "1" if this procedure is not available for this MTF.
9Physical Parameter (physical_parameter-108_1) BOOLEANThis field specifies whether or not this procedure belongs to the physical
parameter class of procedures (procedures that have numeric value results,
such as vital signs, I&O, or patient measurements). Flag is set "0" for No
and "1" for Yes.
10Prompt Label (prompt_label-108_1) STRINGEnter the abbreviated prompt identifier of the procedure for result
entry purpose. e.g. "BP" for blood pressure. Generally, it is suffixed by a ":".
11Special Procedure Type (special_procedure_type-108_1) ENUMERATION This field specifies whether or not this procedure is a consult that can be
processed as an actual order. If this is an ancillary procedure, leave
this field blank. Set this field to "C" (consults) if this procedure is to
be ordered as a consult, rather than an ancillary order type.
 CONSULTS
12Output Title (output_title-108_1) STRINGFree text - field allows 1-30 characters.
13Not Orderable (not_orderable-108_1) BOOLEANEnter "1" if this procedure is not individually orderable.
14Default Cont. Timing (default_cont_timing-108_1) POINTER (Order_Timing-102_2)Field is a pointer to the Order Timing File (#102.2). This field contains
the name of the timing instructions (i.e., BID, QID) associated with this
procedure. Enter data in this field only if the SCHEDULE TYPE field has
been set to 4 (CONTINUOUS).
15Delimiter (delimiter-108_1) STRINGFree text field - enter 1 character
16Standard Entry (standard_entry-108_1) ENUMERATION This field is used to indicate that the procedure is a standard procedure.
Some of the fields are not available for update/change.
 STANDARD ENTRY
17Description (description-108_1) STRINGEnter a detailed description for the procedure. This information is
available during Order Entry when the user enters "?PROCEDURE NAME"
at the "Select CONSULT PROCEDURE" or Select ANCILLARY PROCEDURE" prompt.
18Valid Range (valid_range-108_1) STRINGEnter the valid range or numeric result value for this procedure
in the format of "low-high", eg 20-200. This field is required
if the RESULT TYPE is NUMERIC.
19Unit Of Measure (unit_of_measure-108_1) STRINGEnter the unit of measure if the RESULT TYPE is numeric, eg "kg" for
kilogram.
20Age Group (age_group-108_1) [OBJECT] Multiple field - identified by a selected range.
 
Subclass: Age Group (108_15)
OrderLabelTypeDescription
20.1Upper Limit (upper_limit-108_15) IENThis is the upper age limit of the age group you have selected. If you are
defining the upper limit for adults or ALL then enter 999. If you are defining
the upper limits for infants less than one year old, then enter the decimal
number of the year. (ie .275 = 100 days old).
20.2Age Group (age_group-108_15) STRINGThis is the name of the age group for a set of reference values. It can be
a general term like ADULT, NEWBORN, CHILD, or a specific age range like
2-5yrs.
20.3Normal Range (normal_range-108_15) STRINGEnter the normal range of numeric result value for this procedure, in this
age group. Use the following format: 'low-high' (ie 45-75 ).
21Related Procedures (related_procedures-108_1) [OBJECT] Enter all the related procedures associated with this procedure
for result entry purposes. E.g. for Take Temperature, Take Pulse and
Take Respiration are related procedures.
 
Subclass: Related Procedures (108_13)
OrderLabelTypeDescription

for result entry purposes. For example, for the procedure Take Temperature,
Take Pulse and Take Respiration are related. This field is required if
the RESULT TYPE is (P)ANEL.
21.2Prompt Order (prompt_order-108_13) NUMERICField allows a number between 1 and 999.

must have to 'sign' the order without the doctor's signature.
24Display Clinic Specialty (display_clinic_specialty-108_1) BOOLEANThe Clinic Specialty will be available during Order Entry if set to 'YES'.
25Display Provider Specialty (display_provider_specialty-108_1) BOOLEANThe Provider Specialty will be available during Order Entry if set to 'YES'.
26Display Consulting Clinic (display_consulting_clinic-108_1) BOOLEANThe Consulting Clinic will be available during Order Entry if set to 'YES'.
27Display Consulting Hcp (display_consulting_hcp-108_1) BOOLEANThe Consulting HCP will be available during Order Entry if set to 'YES'.

29Provider Specialty (provider_specialty-108_1) POINTEREnter the specialty of the provider desired for the patient's appointment.
30Scheduled Consulting Clinic (scheduled_consulting_clinic-108_1) [OBJECT] Enter the clinic that is desired for the patient's consultation.
 
Subclass: Scheduled Consulting Clinic (108_18)
OrderLabelTypeDescription

30.2Default Device (default_device-108_18) POINTER (Device-3_5)Consult orders will print to this device at the time that they are scheduled.
31Countersignature (countersignature-108_1) ENUMERATION Set flag to "1" if countersignature is required to order a procedure.
 REQUIRED
32Authorization Key (authorization_key-108_1) POINTER (Security_Key-19_1)Field is a pointer to the Security Key file (#19.1). This field contains
the name of the required security key needed to order the procedure.
33Order Review Required (order_review_required-108_1) BOOLEANField is set "Y" for Yes and "N" for No, indicating whether or not orders
for a procedure must be reviewed.

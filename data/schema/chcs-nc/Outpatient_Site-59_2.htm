Outpatient Site (59_2) Schema Properties:


This file contains records of Outpatient Pharmacy locations and
their parameters.

OrderLabelTypeDescription
1Name (name-59_2) STRINGThis field should be the name of the pharmacy or, if it is
a satellite pharmacy, the name of the satellite.
2Mailing Frank Street Address (mailing_frank_street_address-59_2) STRINGEnter the street address of the site.
3Area Code (area_code-59_2) NUMERICEnter the area code of the site.
4Phone Number (phone_number-59_2) STRINGEnter the phone number of the main pharmacy, preferably that of the Head
Pharmacist.
5Mailing Frank Zip Code (mailing_frank_zip_code-59_2) STRINGEnter the zip code for the site.
6Site Number (site_number-59_2) STRINGEnter an identifying number for the site, with the number being
three to eight characters in length. This number may then be
used to choose a site in the functionality.
7Mailing Frank City (mailing_frank_city-59_2) STRINGEnter the city or town where the site is located or closest to.
8Mailing Frank State (mailing_frank_state-59_2) POINTER (Geographic_Location-5)Enter the state, country, or territory containing this site.
9Closed (closed-59_2) BOOLEANThis field is filled in by the computer, based on whether the
Pharmacy User is opening or closing the Outpatient Pharmacy. The
computer enters "0" for open and "1" for closed.
10Allow Sort On Batch (allow_sort_on_batch-59_2) BOOLEANSet "0" for no or "1" for yes.
11Default For Patient Batches (default_for_patient_batches-59_2) BOOLEANSet to YES if you want this Outpatient Pharmacy Site to be the default
pharmacy at the Select Outpatient Site prompt when users access the Batch
Prescription Entry option. Users can accept the default or select a different
Outpatient Site.

Only one Outpatient Site in the Division can be defined as the Default for
Patient Batches.

***IMPORTANT***

One Outpatient Site in each Division must be defined as the Default for
Patient Batches. If no default is defined, users in that Division cannot
access the Batch Prescription Entry option.
12Parent Site (parent_site-59_2) POINTER (Outpatient_Site-59_2)Enter the name of the pharmacy to which HCP-entered prescriptions
will be re-routed if this pharmacy is closed. This is a required
field.
13Formulary Group (formulary_group-59_2) POINTER (Formulary_Group-59_3)Select the Formulary Group that this Outpatient Pharmacy site
belongs to. The Formulary Group is the group that contains
this site's formulary data definitions.
14Pdts Pharmacy Number (pdts_pharmacy_number-59_2) NUMERICThe NABP format for CMUMPS is NNNNNNC. This is a numeric field with a range
from 0100002 to 9999992. This field contains the number that will be used
by the Pharmacy Data Transaction Service to uniquely identify a
dispensing pharmacy location.
15Pdts Pharmacy Invalid (pdts_pharmacy_invalid-59_2) ENUMERATION This field is used to determine if the Pharmacy Data Transaction Service
Pharmacy Number defined for this Outpatient Site (Dispensing Pharmacy) is
acceptable by the PDTS. The field will be a set of codes and will be set
to 1 if the pharmacy has been rejected by the PDTS with an administrative
Hard edit. Otherwise, the value will be zero or NULL indicating that the
pharmacy number is valid and prescriptions can be ordered from this
dispensing pharmacy. This field need not be populated during file and
table build.
 Valid; Invalid
16Pdts Notification Printer (pdts_notification_printer-59_2) POINTER (Device-3_5)The PDTS Notification Printer is the device that will generate all PDTS
notifications after unavailability occurs. If populated, the system allows
users to send a prescription label to a device without having to batch print
labels. Users will not be permitted to select an outpatient pharmacy site
that does not have the PDTS Notification Printer field populated.
17Npi Id Type Code (npi_id_type_code-59_2) ENUMERATION NPI ID TYPE CODE - this field contains the "ENTITY TYPE CODE" qualifier
associated with the "National Provider Identifier" (NPI - Organizational)
assigned to an Outpatient Site. This value will always be "2" or NULL
This is a non-editable field and is automatically populated when an
NPI is defined for an Outpatient site.
 Individual Provider; Organizational
18Npi Id (npi_id-59_2) NUMERICNPI SERVICE PROVIDER ID - this field is populated with the National
Provider Identifier (NPI ID), Type 2, which uniquely identifies a
health care provider (Organizational - the processing Outpatient
Pharmacy Site) in standard transactions, such as those transmitted
to the PDTS and TPOCS.
19Mtf Division (mtf_division-59_2) POINTER (Medical_Center_Division-40_8)Enter the Division to which this Outpatient Site will be linked.
This is a required field.
20Prescription Number Prefix (prescription_number_prefix-59_2) STRINGEnter a single or double alpha character to be used to designate
the prescriptions from this Outpatient Pharmacy.
21Forward Bulletin Printer (forward_bulletin_printer-59_2) POINTER (Device-3_5)When a prescription is forwarded to you by another Site, If you wish to
recieve notification of the Action, enter the device that you want
the notification printed on.
22Division Parent (division_parent-59_2) BOOLEANThe Divisional Parent is the only site that can be its own parent.
Prescriptions entered via Order Entry from Hospital Locations not defined
as being served by a pharmacy site will be routed to the Divisional Parent.

**IMPORTANT**
It is critical that each Division always have one site defined as the
Divisional Parent.
23Prescription Routing (prescription_routing-59_2) ENUMERATION This parameter controls the routing of prescriptions for a closed
site. Answer with 'DISABLE' if you don't want physician-entered
prescriptions routed to your Parent Site when closed. Answer
with 'ENABLE' if you do want physician-entered prescriptions routed
to your Parent Site when closed.
 DISABLE; ENABLE
24Performing Meprs Code (performing_meprs_code-59_2) POINTER (Meprs_Codes-8119)This MEPRS code is used to report workload performed by the pharmacy. Only a
Pharmacy MEPRS code (beginning with "DA") belonging to the Group DMIS ID of
this pharmacy's division may be entered.

This is a required field. No workload will be reported for sites for
which no PERFORMING MEPRS CODE is entered.
25Vault Used For Narcotics (vault_used_for_narcotics-59_2) POINTER (Controlled_Inventory_Vault-59)If prescriptions for controlled substances are being tracked from
the Narcotic vault, enter the name of the vault from which they
are to be tracked.
26Track Narcotic Prescriptions (track_narcotic_prescriptions-59_2) BOOLEANThis field determines if quantities dispensed for prescriptions
for controlled substances which are stocked in the vault, will be
tracked directly from the vault. Answer "yes" if this Pharmacy
dispenses directly from the Narcotic vault. Answer "no" if this
pharmacy will be reconciling quantities for controlled substances
by issuing them from the narcotic vault.
27Forwarding Site (forwarding_site-59_2) POINTERIf this is multilple site implementation, Define the Site's that you will
accept a prescription forwarded from. Only the Pharmacy Site's that are
defined will be able to forward a prescription to your site.
28Dea Schedule (dea_schedule-59_2) ENUMERATIONIf your site accepts forwarding from other sites, it is assumed that
nonscheduled drugs will be accepted. If your site will also
accept scheduled drugs, the specific legal schedules must be
entered here, or else forwarding will be prohibited.

31Site Label (site_label-59_2) BOOLEANAnswer 'YES' to ENABLE site specific labels for prescriptions.
Additionally, in order to receive site specific labels, both label
template fields must be populated with print templates that have been
compiled into routines. If these fields are not populated, you will
receive the standard CMUMPS label, even if you answer 'YES' to this prompt.
For more specific instructions, see the help text for the individual

32Label #1 Template (label_number1_template-59_2) POINTER (Print_Template-_4)This field should only have an entry if the site has decided to do their
own custom outpatient prescription label. It is important to note that
once the site has defined the print template for the first RX label
they must then use the print template compiler routine (DIPZ) and compile
this print template into a Government Site-specific namespaced routine
such as GSPTLA. The field, LABEL #2 TEMPLATE (prompted as "Template for
Label #2"), must also be populated and compiled for site specific labels
to print.

Also note that in order to get the second label, if one is required the
variable PSCONT must be set to "1" within this print template, using
the function PSCONT (hint: see Print Template PSO-LABEL1).
33Label #2 Template (label_number2_template-59_2) POINTER (Print_Template-_4)This field should only have an entry if the site has decided to do their
own custom outpatient prescription label. It is important to note that
once the site has defined the print template for the continuation RX label
they must then use the print template compiler routine (DIPZ) and compile
this print template into a Government Site-specific namespaced routine
such as GSPTLA1. The field, LABEL #1 TEMPLATE (prompted as "Template for
Label #1") must also be populated and compiled for site specific labels
to print.
34Bar Code Enabled (bar_code_enabled-59_2) BOOLEANThis field enables/disables bar code output on outpatient prescription labels
by site. If this field is set to one (1), then bar code fields on the labels
are enabled, and will be printed on bar code-capable devices. If this field
is zero (0) or NULL (""), then the bar code fields are disabled, and will not
be printed.
35Inactive (inactive-59_2) BOOLEANSites that maintain the Clinical Screening Parameters
CANNOT be inactivated.

Before a site is INACTIVATED, do the following:

1) Set the 'Track Controlled RXS' field to NO.
2) Set the 'Divisional Parent' field to NO.
*It is critical that another site be defined as the Divisional
Parent if the site you are inactivating was the Divisional Parent.
3) Set the 'Default for Patient Batches' field to NO.
*Define another site as the Deault for Patient Batches if the site you
are inactivating was the Default for Patient Batches.
4) Delete the Hospital Locations this site serves in the 'Define
Locations Served' option.
5) Delete this site from the 'Parent' field of any other site.
36Notification Period (notification_period-59_2) STRINGTo disable notification labels, the Dispense Prescription option must be
enabled for the division assigned to the outpatient pharmacy site.
The Notification Period is the maximum number of days between the most
recent label print date of an Rx and the date of an HCP action (modify,
renew, cancel, or hold), on that Rx, that will cause notification labels
to be generated. Notification labels are printed in the pharmacy to notify
pharmacy personnel of the HCP actions described above.

A null entry will imply a Notification Period of 365 days.

Lab Method (8710) Schema Properties:


The Lab Method file provides a means for changing normal ranges without
impacting previously reported results. The file contains the
site/specimen multiple which was formerly part of the Lab Test file.
Rather than creating a new test to reflect a change in method that results
in new reference ranges, multiple methods with their associated reference
ranges can be designated for a test. The methods can be specified as a
default method or can be changed at the time of result entry.

5.6Units (units-8710_01) STRINGUNITS represents a measured quantity, such as grams or a number per
cubic millimeter or microliter. The field length is defined as 1-10
characters. When entering units, be consistent in the use of upper or
lower case for all tests that use the same unit entry.
5.7Delta Check (delta_check-8710_01) POINTER (Execute_Code-62_07)Delta checks are generally used to compare a current result with a previous
result for the same test. The two choices are ABSOLUTE and PERCENT.

The ABSOLUTE type of delta check looks for a change in the result based on
an exact amount defined in the delta value field. The MTF also defines
how many days back the system should check for a delta result. This is
defined in the Lab Site file. This method is most frequently used with
tests like glucose or potassium.

The PERCENT type of delta check looks at the current value and determines
if the result has changed greater than the percentage defined in the
delta value field. This method is commonly used for tests like WBC
or PLT.
5.8Delta Value (delta_value-8710_01) STRINGThis field contains the numeric value chosen for the type of delta check
specified in the `Type of Delta Check' field, either "ABS VALUE" or
"PERCENT". Samples which produce results outside of defined delta check
ranges are appropriately flagged. The delta check flag is not seen on any
reports once the test is certified. It is an internal check for use by lab
personnel.
5.9Default Value (default_value-8710_01) STRINGIf a particular test has the same answer most of the time (i.e., >80% of the
time), entering the common answer in this field will create a "default"
answer or prompt. This prompt will allow the tech to enter the common
answer by simply hitting the return key when entering, editing or
certifying results. If there is no common answer, leave this field blank.
5.10Decimal Places (decimal_places-8710_01) NUMERICEnter the number of decimal places allowed for this result. If no entry is

not issue a warning.

5.12Result High Limit (Absolute) (result_high_limit_absolute-8710_01) NUMERICIf the result type is numeric, enter the highest number possible for the
result. The system will not accept a result higher than the number entered
into this field. Use this value to help control possible typographical
errors input by users.
5.13Result Length (result_length-8710_01) NUMERICThe Result Length field defines the maximum number of characters allowed
for a test result. If the test result type is free text, enter the
maximum number of characters allowed. If the test result type is set of
codes or numeric, skip this field.

5.19Interpretation (interpretation-8710_01) STRINGThis field allows the user to enter a comment or other remarks pertinent

comment will appear on the Interim Report, Patient Inquiry and Cumulative
Report for every test performed on this site/specimen. This remark will
also appear on the test information display. The text may be other
reference range information relating to special conditions.

Use this field to enter reference ranges not related directly to sex and
age. An example is the hormone reference values. Often there are criteria
like menstrual cycle or gestation age associated with the reference
ranges. The reference ranges can be entered here and will always display
with the results.

Subclass: Age Group (8710_04)
OrderLabelTypeDescription
5.20.2.1Upper Limit (upper_limit-8710_04) IENDefine the upper limit for adults or ALL as 999. When defining the upper
limit for infants less than one year old, enter the decimal number portion
of the year. (For example, A baby 100 days old equals .273785 yr.)

If less than 6 decimal places are entered, always round the last digit to
the next highest value. This will ensure that the age calculation for the
patient will fall within the upper limit specified for the age group.

Example:

An age group of 3 DAYS has an upper limit of 3/365.25 = .008213, expressed
to 6 decimal places. If a 6 decimal place entry is made in the field, then
no rounding is necessary. If the upper limit entry is 5 decimal places,
then enter .00822. Note that the 5th decimal place entry is rounded up even
though the numeral in the 6th decimal place is less than 5. If the upper
limit entry is 4 decimal places, then enter .0083. Once again, note that
the 4th decimal place entry is rounded up without regard to the numeral in
the 5th decimal place. Do not make entries having less than 4 decimal
places, as the rounding process may cause the upper limit entry to change
by greater than 1 day.
5.20.2.2Age Group (age_group-8710_04) STRINGThis is the name of the age group for a set of reference values. It can be
a general term such as ADULT, NEWBORN, CHILD, or a specific age range such
as 2-5 yrs. If there is no specific age range, enter ALL. This name is
a label that prints on the display of Lab Test Information.
5.20.2.3Reference Low (reference_low-8710_04) STRINGEnter the lower limit of values for the particular test when performed
on this type of specimen. Any result lower than this entry will be
flagged with an "L" whenever the results are displayed.
Limit the length of any text used so it will fit into the report formats.
Also keep in mind the alert flags defined in the Lab Test file for
sets of codes. Make sure the reference values and the alert flags defined
in the set of codes match.
5.20.2.4Reference High (reference_high-8710_04) STRINGEnter the upper limit or value for the test performed on this type of
specimen. Any result higher than this entry will be flagged with an "H"
whenever the results are displayed. If the reference value is text,
be sure the text length will fit on the report formats.
5.20.2.5Panic Low (panic_low-8710_04) STRINGEnter the low value for this test which designates a "panic value" or
"life threatening" result which would need to be brought to the
technologist's, supervisor's, physician's and/or ward personnel's
attention. When a test value falls below the limit set here, the system
asks for a confirmation step during result entry. When a result displays
through a reporting or inquiry option, the result is followed by a "L*".
5.20.2.6Panic High (panic_high-8710_04) STRINGEnter the high value for this test which designates a "panic value"
result needing confirmation or additional review before the
certification and release of the result to the physician. All of the
information about panic low holds true for panic high except that flags
occur when the test result is higher than the limit set here. When
results are reported or displayed, the result is followed by a "H*".

9Secondary Test (secondary_test-8710) [OBJECT] This multiple contains the fields required to define a potential secondary

Subclass: Secondary Test (8710_08)
OrderLabelTypeDescription
9.1Secondary Test (secondary_test-8710_08) POINTER (Lab_Test-60)This is the name of the test that may be ordered for a secondary order
based on the criteria defined in the Boolean Expression field of the
multiple. The order will be generated for the sample already in the lab.
The process of producing the order, though, is very automatic.
9.2Boolean Expression (boolean_expression-8710_08) STRINGThis is an expression the system will use to determine if the result
entered warrants ordering a secondary test to verify an abnormal result.
This order can be placed automatically by filling in the AUTOMATIC ORDER
field, otherwise, the Lab Tech will be prompted and given the choice whether
or not to order it. The expression can optionally contain an "LRX" at the
beginning. Followed by one of the following <, >, = or [ (contains).

Examples are:

Result Secondary Test Boolean Expression
------ -------------- ------------------
Glu >500 Acetone LRX>500 or >500
Glu <50 Insulin LRX<50 or <50
RPR =R VDRL LRX="R" or ="R"
Bilirubin =P Ictotest LRX["P" or ["P"

* Be careful when using the "[" (contains) sign.
This cannot be used in the case of RPR to ["R". It will
also pick 'NR' and 'WR' because they contain a 'R'.
* When using the = or [ followed by a letter (ie., P for Pos
or N for Neg) the letter must be in " " (quotes).
9.3Requesting Hcp (requesting_hcp-8710_08) POINTER (Provider-6)The lab user must first enter a lab-specific provider into the provider
file. This provider could have a name like "Lab, Generated" and a
provider class of 'Outside Provider' so that the provider will never have
orders that need to be signed and the Patient Lab Inquiry will indicate the
source of the new orders.

Once the new provider is in the provider file, enter it here. The
system will pull the name from this entry when the order is generated.
If a provider is not entered in this field, the system uses the provider
entered in the original order as the requesting HCP.
9.4Requesting Location (requesting_location-8710_08) POINTER (Hospital_Location-44)Enter the requesting location of all laboratory-generated orders. This
will probably be the Lab Section that normally does the test. If this
entry is blank, the system uses the Requesting Location from the
original order.
9.5Meprs Code (meprs_code-8710_08) POINTER (Meprs_Codes-8119)Enter the MEPRS code for the laboratory section. If this entry is blank, the
system uses the code from the original order.
9.6Suppress Report (suppress_report-8710_08) ENUMERATION If this test is requested by the lab user, but the lab does not want the
result displayed to users other than lab personnel, enter a "1" in this
field. The result will not appear on any copy of reports and will not
display during Patient Lab Inquiry for the HCPs.
 SUPPRESSED
9.7Automatic Order (automatic_order-8710_08) BOOLEANYes in this field will automatically place a secondary order if the
BOOLEAN XP is "TRUE".

Enter "YES" in this field if you want a secondary order to be automatically
ordered without prompting the user to place the order. A "NO" or null
in this field will prompt the user to place the order if the BOOLEAN XP is
"TRUE".
9.8Automatic Accession (automatic_accession-8710_08) BOOLEANYes in this field will automatically accession and log-in tests ordered
through secondary orders.

Enter "YES" in this field if you want the order to be automatically
accessioned. For tests performed in the same accession area as the
primary test, the order will be added to the current accession number.
For tests performed outside of the accession area in which the primary
test is performed or the secondary test requires a unique accession,
a new accession number will be generated.

If the Collection Sample of the Secondary Test is different than the
Collection Sample of the Primary Test, the order will not be
accessioned.
10Lab Test Used In Calculation (lab_test_used_in_calculation-8710) [OBJECT] This multiple contains the names of the tests used in the calculation of the
calculated tests. These tests are used in the Formula field.
 
Subclass: Lab Test Used In Calculation (8710_09)
OrderLabelTypeDescription
10.1Lab Test Used In Calculation (lab_test_used_in_calculation-8710_09) POINTER (Lab_Test-60)Enter the name of the lab tests to be used to calculate the result for this
test. The lab tests must be defined for the MTF. The tests used in the
calculation are only entered into this multiple if the test being defined is
the calculated test. For example: If the test being defined is CALC. LDL
then CHOLESTEROL, TRIGLYCERIDE, and HDL would be entered as the lab tests
used in the calculation. The system will prompt the user to enter results
for the lab tests used in the calculation, then will return a result based
on the calculation formula.
10.2Variable (variable-8710_09) STRINGEnter a variable name for the lab tests to be used in the calculation using
the format of the letter C followed by the number of the variable in
parentheses [example: C(1) for the first variable; C(2) for the second].
Each test entered must have a unique variable number. These variable names
will be used in the formula for calculating a calculated-type test. See the
FORMULA field for further explanation.
11Formula (formula-8710) STRINGEnter the formula for the calculation. Use the operators +, -, / and *
for addition, subtraction, division and multiplication respectively.
Use the variables created in the "LAB TEST USED IN CALCULATION" multiple.
Example: T7=T4+T3 The variables are T4=C(1) and T3=C(2). The formula
entered for this calculation would be C(1)+C(2).

The system will read the calculations from left to right. If the formula
requires a specific portion to be calculated separately, enclose that
section in parentheses. In the example C(1)*100/(100+C(2)), if the
parentheses were not included the system will calculate C(1)*100 then
divide by 100 and then add C(2). With the parentheses the system will
calculate 100+C(2) then divide it into C(1)*100.

temporarily suspended pending some action which must take place, such
as a nursing signature or an appointment made. If inactive, this
particular order has either been resulted, expired or cancelled/discon-

 INACTIVE; ACTIVE; PENDING
12Active Status (active_status-103_1) ENUMERATION This field further defines the status of an ACTIVE order. A FULLY ACTIVE
order means that there are no conditions placed on the order's activity.
SIGNATURE ABSENT means that the HCP and/or COUNTERSIGNING HCP have not
yet signed the order, although the order is active because a nurse

the display screen.
 FULLY ACTIVE; SIGNATURE ABSENT
13Inactive Status (inactive_status-103_1) ENUMERATION This flag further defines the status of an inactive order. If RESULTED,
the order has had all tasks resulted. EXPLICITLY DISCONTINUED means
that an action was taken by a provider to stop this order. TRANSFERRED
is the inactive status created when a patient is transferred within
the MTF. At that point, all orders are 'discontinued' but the provider
has an opportunity to later mark which of these orders he would like
to 'reactivate'. This reactivation actually results in new orders
being generated. SIGNATURE MISSING means that an order has been
discontinued but still needs a signature or countersignature. It
will continue to show up on the display screen until signed. If a
provider cancels an order requiring a signature, and he is qualified
to sign that order, the signing takes place automatically and this
status is avoided.
 RESULTED; DISCONTINUED; TRANSFERRED; SIGNATURE MISSING; RESULTED-NO SIG
14Pending Signature (pending_signature-103_1) ENUMERATION This pending flag indicates that an order has not been activated
because either a nurse or HCP signature has not yet been obtained.
 PENDING SIGNATURE
15Pending Appointment (pending_appointment-103_1) ENUMERATION This pending flag indicates that an order is suspended while awaiting
completion of the scheduling process. Because one order may generate
several tasks requiring scheduling, this flag is set when NONE of the
remaining tasks have been scheduled - in other words, when there is
a potential hold up of the process as a result of schedules not being
completed. Once the first task is scheduled, the order is active unless
the first task is completed and the next task is not yet scheduled.
 PENDING APPOINTMENT
16Pending Activation (pending_activation-103_1) ENUMERATION This pending flag indicates when an order has been placed on hold.
This status changes either when the user activates the order or
when the ACTIVATION DATE arrives and the order is taken out of
suspense automatically. In this instance, the order is then placed
in PENDING REVIEW (see next field).
 PENDING ACTIVATION
17Pending Review (pending_review-103_1) ENUMERATION This pending flag is set when an order on HOLD (PENDING ACTIVATION)
reaches the activation date. Rather than automatically activate the
order, this status alerts the user that the order is due to be activated
after he/she has reviewed it for its appropriateness at this time.
 PENDING REVIEW
18Activation Date (activation_date-103_1) DATE-TIMEWhen an order is put in suspense (as for preadmission orders), the user

collection (how quickly should a specimen be obtained). The two

the Order Priority file in which the MTF can place additional
priority levels if it wishes.
If the order has STAT priority, the system will be monitering the
time to acknowledge the order and then the time to result the
order. Times exceeding MTF defined intervals will be tracked for
QA.

Clinical User (8820) Schema Properties:


This file contains all the user-specific data related to the Desktop,
and other Clinical Interface functions. The entries point to the
User file, and this file may be thought of as an extension of it.
There are numerous fields that deal with user preferences. There
are also 4 subfiles, containing multiple entries for View Text,
and Patient Lists, Patient Text & Topics, and Macro Commands. There
are multiple cross-references which contain dynamic data related
to Desktop functions, such as the last Desktop list, what user is
currently in the Desktop, what user has locked their desktop, etc.
Each user accessible field has help text associated with it, that
can be reviewed from within the Desktop via the Help action.

OrderLabelTypeDescription
1Name (name-8820) POINTER (User-3)This is the name of the Clinical User, which is the same as the

2Control Mode (control_mode-8820) ENUMERATION This field is used by Desktop to set up the initial Control Mode.
If it is not set or set to COMMAND LINE", then the user will
Select Actions by typing a command, then pressing Enter.
If it is set to ACTION BAR, then the user will Select Actions by
moving the cursor over the items, or pressing the first character
of the item. If there is more than one item with the same first
character, then the cursor will move among them, and the user
will have to press Return at the desired item.
This field only determines the INITIAL Control Mode, when entering
Desktop. The user may switch back and forth once in Desktop.
 COMMAND LINE; ACTION BAR
3Restore (restore-8820) ENUMERATION This field is used by Desktop to set up the initial Desktop display.
If it is not set or set to "NO", then the user will be asked to
select patients each time the Desktop is entered.
If it is set to "SELF", then their last Desktop will be restored.
If it is set to "OTHER", then the user will be asked to select a
Desktop to restore, either their own or another user who has set
their Desktop to Public Access.
 NO; SELF; OTHER
4Access (access-8820) ENUMERATION This field determines access to the user's Desktop by other users.
If it is not set or set to "PRIVATE", then no other user may use
Desktop as a source of patients.
If it is set to "PUBLIC", then another user may use this Desktop
as a source of patients for their Desktop.
 PRIVATE; PUBLIC
5Initial Display (initial_display-8820) ENUMERATION This field determines the initial patient Display on entry into
the Desktop.
If it is not set or set to "ALL", then all the patients will be
displayed in alphabetical order by patient name.
If it is set to "WARD", then all the patients will be displayed
sorted by Room-Bed, grouped by ward.
If it is set to "IN-PT", then all of the in-patients will be
displayed alphabetically by name.
If it is set to "OUT-PT", then all of the out-patients will be
displayed alphabetically by name.
If it is set to "CURRENT ID", then all of the patients will be
displayed sorted first by the current identifier.
If any subgrouping is selected, and there are no patients in it,
then the display will revert to "ALL".
 ALL; WARD; IN-PT; OUT-PT; CURRENT ID
6List Save (list_save-8820) ENUMERATION This field determines what happens if the user saves a list of items
to a permanent user file which already exists.
If it is not set or set to "ASK", then the user will be prompted
whether to replace the existing file with the new items, or
merge the newest items with the existing ones.
If it is set to "MERGE", then the user will not be prompted, but
the new items will automatically be merged with the existing ones.
If it is set to "REPLACE", then the user will not be prompted, but
the old items will be deleted, and the new items stored under the
chosen List name.
 ASK; MERGE; REPLACE
7Wait Time (wait_time-8820) NUMERICIf you have set your Desktop to restore on entry, then this field
will determine how long to wait for you to abort the restore.
If it is not set or set to '0', then you will not be able to abort.
If it is set to -1, then it will wait the normal time-out period
for you to respond.
If it is set to a positive number, then it will wait that many
seconds, before continuing.
The main purpose of this field is to allow you to set your Desktop
to restore, and still let you abort the restore procedure under
specific circumstances, without having to change your preferences.
8Purge Desktop (purge_desktop-8820) ENUMERATION This field determines what happens when the user Purges their Desktop
If it is not set or set to "ASK-"..., then the user will be prompted
whether to confirm the purge, defaulting as specified by the user
If it is set to IMMEDIATE then the user will not be prompted, and the
Desktop will be purged without confirmation.
 ASK-'NO'(DEFAULT); ASK-'YES'(DEFAULT); IMMEDIATE
9Patient Identifiers (patient_identifiers-8820) ENUMERATION This field determines whether the Desktop patients will each be
identified with their Room-bed, or by the user's customized IDs.
If it is not set or set to "ROOM-BED", then the right side of the
display will show the patient's Room-bed, if any.
If it is set to "USER-DEFINED", then for each of 5 columns defined
by the user, a user-defined symbol will be placed in the column,
if the patient has the user-defined characteristic.
If it is set to "BIRTH DATE", then the right side of the display

If it is set to "CLINIC APPOINTMENT", then the right side of the

and date range.

display will only be updated if the user selects the Update action,
or adds or deletes patients from the Desktop.
 BIRTH DATE; ROOM-BED; USER DEFINED; CLINIC APPOINTMENT

41Patient List (patient_list-8820) [OBJECT] This subfile contains the Patient List entries. Each entry has a Name,
a Creation date, an Access field, a word processing Description, and
multiple Patient entries.
The Patient Lists may be created interactively, or automatically by
the Patient List Addend Desk Action.
To create a new list interactively, the user should use PLE to get
to the screen showing the current (if any) Patient Lists. Place
the cursor below the last list entry, and type the name of the new
list. The system will ask if the user is adding a new list, and
after confirmation, present the next screen for completion.
To add a new patient to the list, place the cursor below the last
patient, and type the name of the desired patient. The system
will then add the patient to the list, and present the next
screen for completion.
To create a list automatically, user the PL+ to save the selected
Desktop patients to this subfile. After the patients are chosen,
the system will prompt the user for a Patient List name. If a new
name is chosen, the system will save these patients under that name
after confirmation that the user is creating a new list. If the
user chooses an existing name, the system will either ask, merge,
or replace the existing list, based on the user's preferences.

42Patient Text (patient_text-8820) [OBJECT] This subfile contains the Patient Text entries. Each entry has a
patient, an Entry date, and multiple Topics.
The Patient Texts may be created interactively, automatically by
pressing the F9 key when browsing the patients in the Desktop, by
the PTSE (Patient Text Selected Edit) Desk Action, or the PLTE
Patient List to Text Edit) Desk Action.
To create a new entry interactively, the user should use PTE to get
to the screen showing the current (if any) Patient Texts. Place
the cursor below the last patient entry, and type the name of the
new patient. The system will look up the patient, then ask if you
are adding a new entry. Upon confirmation, you will be presented
with the next screen, which will allow entry/editing of Topics.
To add a new Topic to the Patient Text, place the cursor below the
last Topic, and type the name of the new Topic. Upon confirmation,
the system will then add the topic to the list, and present the
next screen for completion.
To add a patient automatically, browse the patients in your Desktop,
and when the Cursor is to the left of the desired Patient, press the
F9 key. You will then be in the screen for adding/editing Topics.
To add/edit Topics on multiple patients, use the PTS action. After
the Desktop patients are selected, the system will cycle through the
list asking for confirmation after the 2nd patient. Upon confirmation
you will be presented with the screen for adding/editing Topics.

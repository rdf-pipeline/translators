Lab Test (60) Schema Properties:


This file holds the specific site information related to each laboratory
test or laboratory panel. Entries may be added or modified, but none of the
supplied entries should be deleted. The file defines specific information
for each test such as synonyms, accession area, collection sample(s), print
order, CAP/CPT codes and methods unique to the test. Some pieces of data can
be defined for each work element such as test cost, lab print name and order,
overdue criteria, and the result optional flag.

OrderLabelTypeDescription
1Name (name-60) STRING|NOWRAP|
Enter a new or existing test/panel name (full or partial) to edit.

Entry at this prompt allows the user to enter a new non-standard test/panel
or to edit fields for existing tests.

Most existing tests are standardized and cannot be entered into the system
by users. Standard tests have been pre-approved for use at all Department
of Defense facilities.

The user can enter certain non-standard tests that must be Panel or QC
ONLY. In addition, a limited number of Miscellaneous tests are available on
the system to be re-defined as non-standard tests. Re-defined Miscellaneous
tests are always Single tests.

QC ONLY tests are used internally within the laboratory and are ordered
only on Quality Control specimens using the laboratory QC options. Tests
designated as QC ONLY may not be changed to non-QC ONLY.

To re-define a Miscellaneous test for use at this site, select the name of
a pre-defined Miscellaneous test. The user will be required to modify the
Miscellaneous test's name and specify its subscript. When the modifications
to a Miscellaneous test are filed, a mail bulletin will be triggered
notifying members of a mail group that a new Miscellaneous lab test has
been re-defined. The test will be evaluated for future standardization. The
re-defined Miscellaneous test will be non-standard and orders for the test
cannot be transmitted electronically to outside performing facilities.
2Sensitive Results (sensitive_results-60) BOOLEAN|NOWRAP|
Enter "YES" in this field if the results of this test are sensitive
and should be routinely suppressed. The result will be displayed as
"REQUESTING HCP ONLY" unless the user is assigned the "LRSENSITIVE"
key or is the requesting HCP.

The results will be suppressed in the options: Lab Patient Inquiry,
Patient Episode Cumulative, Doctors Cumulative, and Interim report.
The Inpatient and Outpatient cumulative will always report sensitive
results as "REQUESTING HCP ONLY". The lab prints the Sensitive Result
Report in order to provide a chart copy of the result.

If sensitive tests are referred to the laboratory, results will print
on the Referral Report for Sensitive Results. This report and the
Sensitive Result Report are printed in the lab so the results can
be handled more discretely.

Laboratory users must have the LRSENSLAB key in order to process
sensitive tests after log in.

12Type (type-60) ENUMERATION B - (Both) means test can be both ordered and resulted/reported.
C - (Calculated) means test result is automatically calculated by the system.
I - (Input) means test can be ordered, but not resulted/reported by itself.
M - (Miscellaneous) specifies test that can be re-defined for local use.
N - (Neither) means test can no longer be ordered and/or resulted.
O - (Output) means test can be resulted/reported, but not ordered.

The entry in this field determines how the system handles a test for
ordering, resulting, and reporting.

Glucose, TSH, Retic, Occult Blood, etc. are examples of tests that normally
have the "Type" field set to BOTH (meaning these tests can be both ordered
and resulted/reported individually). This represents the majority of tests
in the system.

An example of a CALCULATED test is Calc. LDL. Results for the Calc. LDL
test are automatically calculated by the system from the results of the
TRIGLYCERIDES, CHOLESTEROL, and HDL tests.

An example of an INPUT test is a CBC. An order is placed for a CBC, but
results are entered and reported for the individual tests of the CBC (i.e.
WBC, RBC, HGB, etc). Single tests do not usually have the "Type" field set
to INPUT.

Tests designated as MISCELLANEOUS are used to allow the site to set up new
tests as necessary. Tests already assigned as another type cannot be
changed to miscellaneous. Miscellaneous tests are pre-defined in the system
and cannot be added by the site. Whenever a site re-defines a miscellaneous
test, a mail bulletin is generated to a group authorized to evaluate the
test for inclusion as a DOD Standard Lab Test.

Tests can be made unorderable by changing the "Type" field to NEITHER. If
there is no need to order, edit, or display a test, it may be inactivated
by setting the "Inactive Flag" field to YES. Once a test has been made
inactive, it cannot be re-activated. Deleting entries from the Lab Test
file is not allowed, since database integrity will be compromised.

Components of a panel are examples of tests that normally have the "Type"
field set to OUTPUT. Specific examples are tests such as Appearance,
Turbidity, Urine Glucose, Crystals, etc. that are always ordered as part of
a Urinalysis panel.
 BOTH; CALCULATED; INPUT; MISCELLANEOUS; NEITHER; OUTPUT
13Subscript (subscript-60) ENUMERATION Enter a subscript for the Lab Test entry.

Examples are CH for Clinical Chemistry, BT for Bacteriology, and SP for
Surgical Pathology. Each subscript dictates a unique set of order and
result entry requirements for various lab tests.

The Clinical Chemistry subscript allows the user to enter a single result
for each test. Use CH for most laboratory tests including Chemistry,
Hematology, Immunology, and Coagulation tests. New tests with a CH
subscript can be added to the system only as panel tests or QC ONLY tests.

The subscripts for Bacteriology, Parasitology, Mycobacterium, Mycology, and
Virology comprise the Microbiology test group. Each subscript provides the
user with a specialized result entry screen which is determined by the
Micro Screen field designated in the Lab Test file. New tests with a
Microbiology subscript can be added to the system only as panel tests or QC
ONLY tests.

The subscripts for Autopsy, Bone Marrow, Cytology Gynecological, Cytology
Non-Gynecological, and Surgical Pathology comprise the Anatomic Pathology
test group. Each subscript provides the user with a specialized result
entry screen. New tests cannot be added with an Anatomic Pathology
subscript.

A standard set of tests with the Blood Bank (BB) subscript have been added
to the Lab Test file as part of the interface between CMUMPS and DBSS
(Defense Blood Standard System). New blood bank tests may not be added to
the Lab Test file.
 CLINICAL CHEMISTRY; BACTERIOLOGY; PARASITOLOGY; MYCOBACTERIUM; MYCOLOGY; VIROLOGY; SURGICAL PATHOLOGY; AUTOPSY; BONE MARROW; CYTOLOGY GYN; CYTOLOGY NON-GYN; VETERINARY; DENTAL; BLOOD BANK
14Qc Only (qc_only-60) BOOLEANEnter YES if this test is for QC purposes only, NO if test is a panel, or
"^" to exit.

If a single test, enter "YES" to designate test as QC ONLY. Single tests
that are not used strictly for quality control purposes cannot be added to
this file.

If a panel test, enter '^' to delete test or 'NO' to add panel test as a
non-QC test. Panel tests cannot be designated for quality control purposes,
and therefore should have this field set to NO when being defined.

QC ONLY tests may only be used internally within the laboratory and may
only be ordered on Quality Control specimens using the laboratory QC
options.

Tests that are QC ONLY will usually be done in the Microbiology areas
(Bacteriology, Parasitology, Mycology, Mycobacteriology, Virology) and
Clinical Chemistry. An example of a test that might be defined this way is
AMPICILLIN when used by the micro accession area to do quality control on
antibiotics.
